Description and Overview¶

Geocoding is the process of turning an address into a latitude and longitude pair.

Service updates¶

Geocoding service was updated in December, 2010.

HTTP Request Structure¶

The Geocoding HTTP API exposes the method "find" as shown by the following URL:

http://geocoding.cloudmade.com/YOUR-API-KEY/geocoding/v2/find.output_type?query=search_query&other_parameters

YOUR-API-KEY (required) - get it at User Profile page.

search_query - if present, specifies an address to geocode, or a keyword used to search for objects that contain this keyword in their names.

other_parameters - all parameters are optional and are listed in Parameters table.

Note: You can add Geocoding service to you application, examples and explanations can be found on web and iPhone SDK examples pages.

Query¶

If the query represents an address, it can be either in a free form format (for example, query=Oxford+st,+London,+UK), or specified as a set of address parts separated by semicolon. The following address parts are recognized: poi, house, street, city, zipcode or postcode, county, country. For example, query=street:Oxford street;city:London;country:UK.

Query Note:
  • The Search is case-insensitive
  • Search word should match the whole word from the database
  • Wildcards are not currently supported
  • Spaces in the query of search words will be converted to '+' signs in the request

Note: Please note that the Geocoding HTTP API expects UTF-8 encoding. Most browsers (except Chrome) do not encode query parameters as UTF-8, so you will need to adjust their settings if you want to test names that contain characters such as å, ü, Cyrillic characters and other special characters. In Firefox this is done by opening the about:config URL and setting network.standard-url.encode-query-utf8 = true

Geocoding responses¶

Output types can be in:
  • ".js" for JSON

    http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.js?query=133+Fleet+street,+London,+UK

  • ".geojs" for GeoJSON

    http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.geojs?query=133+Fleet+street,+London,+UK

  • ".plist" for Property list

    http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.plist?query=133+Fleet+street,+London,+UK

  • ".html" to render search results on the map (useful for debugging purposes)

    http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?query=133+Fleet+street,+London,+UK

Release Versions¶

Geocoder features are released in 3 stages - Alpha, Beta and Production versions. Each of these versions is graded based on its quality level. Understanding how a service and feature is graded will help you choose features that are right for your application.

  • Alpha - cutting edge, will contain bugs and incomplete features
  • Beta – a generally mature services, but may contain bugs or give unexpected results
  • Production - a mature service with complete features sets and no known bugs

Note: We do not recommend using Alpha or Beta versions of the Geocoding HTTP API in production quality applications.

Note: The results returned by a Geocoding search will vary depending on the quality and coverage of data for a particular region. The standard Geocoding HTTP API uses a combination of different data sources to create a dataset and service that is free for you to use. We also offer a paid-for Geocoding HTTP API that uses premium data. To learn more about this service, contact our Developer Sales Team.

Freeform search (Not released yet) ¶

Allows a user to search for an address using a freeform text string.

In this method, the geocoding request is built in the following way:
http://geocoding.cloudmade.com/YOUR-API-KEY/geocoding/v2/find.output_type?query=SEARCH PHRASE&other_parameters

Additional parameters can be found in the Parameters table.

Free form address search ¶

Allows you to find any address present in CloudMade’s databases. Address coverage varies by region and all address search services are rated “Alpha” services. If you require address level geoocding, a paid-for service is available in most regions. You can contact our Developer Sales Team to find out more.

Address search requests should be stated as 133 Fleet street, London, UK UK, London, Fleet street, 133; 133 Fleet street, London UK

http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?query=133+Fleet+street,+London,+UK

Free form POI search ¶

Used for POI search where an end user can input a POI search request in any way.
POI search request can be formulated as Zoo, Belin, Germany, cafe, Paris, France or even Ukraine Kyiv hospital

http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?query=Zoo,+Berlin,+Germany

The list of POI types that can be found with CloudMade Geocoding is available at Object types page.

Postcode free form search ¶

Allows to find areas, cities, buildings that belong to a specific postcode or zipcode (for USA).

Postcode search request can be built as 90210, USA or even YO31 1EB

http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?query=90210

Reverse Geocoding ¶

Reverse Geocoding finds an object that is closest to a certain point or object.

http://geocoding.cloudmade.com/YOUR-API-KEY/geocoding/v2/find.html?around=object&object_type=OBJECT_TYPE

YOUR-API-KEY - get it at User Profile page.
object - it can be 'lat,lon' or any other obejct around which the search should be fulfilled.
object_type - types of objects to look for, all the possible types can be found on Object types page.

All other parameters can be found in the Parameters table.

Reverse address Geocoding ¶

This method allows you to find the address closest to a certain point.

Reverse address search request can be built as closest address to a point or closest address to a point (51.51558,-0.141449) not more than 50 meters

http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?object_type=address&around=51.51558,-0.141449

Reverse POI Geocoding ¶

This method allows to search for places of interest around certain point. Can be used in location based applications, e.g. find the nearest cafes to my current location using GPS coorditanes, then build the route to it.

All the possible types can be found on Object types page.

Reverse POI search request can be built as 5 cafes closest to a point (51.51558,-0.141449) not more than 500 meters or closest road to a point

http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?object_type=cafe&around=51.51558,-0.141449&results=5&distance=closest

Reverse Objects Geocoding ¶

This method allows to find objects that are closest to the previously found objects, e.g. closest cafe to Oxford Street, London, UK.

Reverse POI search request can be built as closest 5 cafes to Oxford Street, London, UK not more than 500 meters or closest bus stops to Karlstrasse, Berlin

http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?object_type=cafe&around=Oxford%20street,%20London,%20UK&results=5&distance=500

Note: there are some search requests that are very similar, but they return very different search results. The difference appears if such parameters as 'object_type' and 'query' OR 'object_type' and 'around' are used.
http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?object_type=bus_stop&query=Oxford,%20London
'object_type' and 'query' returns: all bus stops called "Oxford, London"
http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?object_type=bus_stop&around=Oxford,%20London
'object_type' and 'around' returns: all bus stops near "Oxford, London"

Structured search ¶

Finds geo objects by structured query.
Query should contain key/value pairs separated by ";", key should be separated from value by ":". Key should be one of the following: country, county, city, street, house, poi.

http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?find.html?query=country:name;county:name;city:name;street:name;house:name;poi:name

All parameters are optional.

As additional parameter bbox - bounding box of area, tells where to look for objects (format "southern_latitude,western_longitude,northern_latitude,eastern_longitude").

http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?query=street:Oxford+street&bbox=51.36688,-0.35239,51.64718,0.09677

Structured Address search ¶

Structured search in word request looks the same as free form search, both are formed as 133 Fleet street, London, UK , Germany, Berlin, Baseler Strasse, 40 , London, UK, SE1 , Los Angeles, USA, 90210":http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?query=city:Los%20Angeles;country:USA;zipcode:90210

http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?query=house:133;street:Fleet+street;city:London;country:UK

Structured street and city search ¶

The search results and search request words are the same - Fleet street, London, UK , Germany, Berlin, Baseler Strasse

http://geocoding.cloudmade.com/8ee2a50541944fb9bcedded5165f09d9/geocoding/v2/find.html?query=street:Fleet+street;city:London;country:UK

Parameters¶

Parameter Explanation Default
results Number of results to return. If results <100 then response.found shows how many objects have been found, but not more then 100. If results >100 then response.found is how many objects returned. 10
skip Number of results to skip from the beginning. 0
return_geometry Set it to true if you want geometry included in search results. false
return_location Set it to true if you do want location information, like road, city, county, country, postcode, in returned results. false
bbox Bounding box of the search area. Format: southern_latitude,western_longitude,northern_latitude, eastern_longitude. Cannot be used together with around / distance.
around Central point of the search area. Used together with distance. around must be an EPSG:4326 coordinate ("latitude,longitude") or an address. Cannot be used together with bbox. If used together with a non-empty search_query, only one may specify an address.
distance Radius of the search area. Distance is specified in meters from the central point. Special value closest limits search results to one, closest to the central point of the search area. 100 meters
bbox_only Used only if bbox is specified. If set to false, the geocoder will return results from the full database, but still ranking results from within the specified bbox higher, otherwise only results from within the specified bbox will be returned. true
object_type Limits search results to a specific object type. Full list of object types can be found here. If around / distance is specified, the default is address.
callback Ability to 'wrap' geocoding response into function, use cmGeocodeCallback value to do it, here is an example
return_parent Set it to true if you do want any high level result like road, city, county, country in returned results if low level does not exist. For example road will be returned if poi is not exist in query "supermarker, oxford street, london". true
along Find objects from Objects list along a line.
